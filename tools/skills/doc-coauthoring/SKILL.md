# Doc Co-Authoring Workflow

Guide users through collaborative document creation via three stages: Context Gathering, Refinement & Structure, and Reader Testing.

## When to Use

- User mentions: "write a doc", "draft a proposal", "create a spec"
- User mentions doc types: "PRD", "design doc", "decision doc", "RFC"

## Stage 1: Context Gathering

Ask for meta-context:
1. What type of document is this?
2. Who is the primary audience?
3. What is the desired impact?
4. Is there a template to follow?

Then encourage info dumping of all relevant context.

## Stage 2: Refinement & Structure

For each section:
1. Ask clarifying questions
2. Brainstorm 5-20 options
3. User selects what to keep/remove
4. Draft the section
5. Iterate until satisfied

## Stage 3: Reader Testing

1. Predict what questions readers might ask
2. Test document against those questions
3. Fix any gaps identified
